- [API server startup](api-server-startup.md) — takes 50-60s to build+start; use `artifacts/api-server: API Server` managed workflow; verify with `curl localhost:3000/api/healthz`
- [Feed pagination pattern](feed-pagination.md) — cursor uses `_id` for both sort and filter to avoid ordering mismatch; sort by `_id: -1`, cursor filter `_id < ObjectId(before)`
- [Supabase realtime DMs](supabase-realtime.md) — useMessages hook listens for broadcast `new_message` AND postgres_changes; server broadcasts on `messages:${id}` after insert
- [Google OAuth email confirm](google-oauth-email.md) — must call confirmUserEmail(supabaseId) after new Google signup or Supabase auto-bans for unconfirmed email; ensureSupabaseInitialized must be awaited first
- [Post visibility](post-visibility.md) — posts were only saved to localStorage; api.posts.create() must be called in handleNewPost to persist to MongoDB for other users to see
- [Supabase Node 20 WebSocket](supabase-node20-ws.md) — Supabase Realtime crashes on Node 20 without ws package; must pass `realtime: { transport: ws }` to createClient; ws package must be installed
- [Notification broadcast](notification-broadcast.md) — persistAndBroadcast() in friends.ts saves to MongoDB notifications collection THEN broadcasts; broadcastNotification() alone never persists so polling finds nothing
- [Media storage](media-storage.md) — media.ts uses Supabase Storage bucket "media" (public); multer memoryStorage + upload buffer; PostComposer uploads base64/blob URLs before posting
- [Messaging vs friend policy](messaging-policy.md) — messagingPolicy (everyone/friends-only) is separate from friendPolicy (open/request-only); messages.ts checks messagingPolicy with fallback to friendPolicy for legacy users
- [useMessages hook signature](usemessages-signature.md) — useMessages(conversationId) takes only 1 arg; sendMessage(content: string, type?) takes a string not an object; getOrCreateConversation takes a ConversationInitData object not two strings
- [AppPhase values](appphase-values.md) — suspension routes to 'account-suspended', ban to 'banned'; 'suspended' is NOT a valid AppPhase and will cause a TS error
- [Report/suspension system](report-suspension.md) — engagement.ts POST /:postId/report dedupes per reporter; increments reportCount on author; suspends at 5+ reports (separate from addStrike 3-strike security system)
- [Signin abuse detection](signin-abuse.md) — auth.ts email signin tracks signinHistory (rolling 20); 6+ signins in 10min triggers addStrike('rapid_signin_cycling'); suspended/banned users blocked at signin with 403
- [Edit tool regex corruption](edit-tool-regex-risk.md) — Edit tool corrupts files when old_string contains regex special chars or when new_string arg has capture group refs; always use WriteFile or Python surgery for files with regex patterns; never use Edit on friends.ts or auth.ts with regex content
- [auth.ts structure](auth-ts-structure.md) — auth.ts had duplicate profile/update, tier/update, and profile/:username routes from prior bad edits; also has phone search endpoint at GET /search-by-phone
- [New Chat sheet pattern](new-chat-sheet.md) — NewChatSheet.tsx opens as a full-screen slide-up from ChatTab; calls api.messages.getOrCreateConversation then setActiveDmConv; uses /api/friends/:username for friends list
- [Bitmoji sticker system](bitmoji-sticker-system.md) — BitmojiAvatar.tsx uses DiceBear avataaars v9 (Snapchat-like); stickers encoded as [BITMOJI:moodId:JSON] in message content; avataaars params: skinColor,top,hairColor,eyes,eyebrow,mouth,facialHair,clotheType,clotheColor,accessories
- [ObjectId guard pattern](objectid-guard.md) — always guard new ObjectId(id) with isValidObjectId check (24-char hex test) before calling; fake/ad IDs like "ad-1" cause BSONError crashes in engagement routes
- [Friend system MongoDB](friend-system-mongodb.md) — friends.ts fully rewritten to MongoDB (friend_requests + friends collections); persistAndBroadcast() saves to notifications collection AND broadcasts; userId must match frontend: supabaseId || username (never _id.toString())
- [Phone save and search](phone-save-search.md) — PATCH /users/:username/phone uses case-insensitive regex match; phone normalized (spaces stripped) on save; search uses last-10-digit regex for flexible format matching; phone index in MongoDB
- [Performance setup](performance-setup.md) — express-rate-limit installed (300 req/min general, 30 req/15min auth); compression middleware; MongoDB pool maxPoolSize:100; trust proxy set for rate limiting behind Vercel
- [Realtime messages RLS](realtime-messages-rls.md) — Supabase messages table has RLS enabled but no policies so postgres_changes subscription fires nothing; fix: server broadcasts on `messages:${convId}` channel after insert; client listens for both broadcast new_message AND postgres_changes
